## 🔄 修改文件

### **application.yml** - 通用配置（已优化）
- ✅ 改为通用基础配置
- ✅ 使用环境变量占位符
- ✅ 支持配置覆盖
- ✅ 添加详细注释

**主要变化:**
```yaml
# 之前: 硬编码配置
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/postgres
    username: postgres
    password: password

# 现在: 使用环境变量和默认值
spring:
  datasource:
    url: ${SPRING_DATASOURCE_URL:jdbc:postgresql://localhost:5432/banking_db}
    username: ${SPRING_DATASOURCE_USERNAME:postgres}
    password: ${SPRING_DATASOURCE_PASSWORD:password}
```

## 🎯 如何使用

### 场景 1: 本地开发

#### 前置条件
1. 安装 PostgreSQL
   ```bash
   # macOS
   brew install postgresql@15
   brew services start postgresql@15
   
   # Ubuntu/Debian
   sudo apt-get install postgresql
   sudo systemctl start postgresql
   ```

2. 创建数据库
   ```bash
   psql -U postgres -c "CREATE DATABASE banking_db;"
   ```

#### 启动应用

**方法 A: 使用 Gradle（推荐）**
```bash
./gradlew bootRun --args='--spring.profiles.active=local'
```

**方法 B: 使用环境变量**
```bash
export SPRING_PROFILES_ACTIVE=local
./gradlew bootRun
```

**方法 C: 使用 IntelliJ IDEA**
1. Run → Edit Configurations
2. VM options: `-Dspring.profiles.active=local`
3. 点击 Run

#### 验证
```bash
curl http://localhost:8080/api/accounts
```

预期响应:
```json
{
  "success": true,
  "message": "查询成功",
  "data": [],
  "timestamp": "2024-12-23T10:30:00"
}
```

### 场景 2: 部署到 EC2

#### Step 1: 创建 AWS RDS

1. 登录 AWS 控制台
2. 创建 PostgreSQL RDS 实例
3. 配置安全组允许 EC2 访问
4. 记录连接信息：
   - Endpoint: `banking-rds.xxxxxx.region.rds.amazonaws.com`
   - Port: `5432`
   - Username: `postgres`
   - Password: `your_password`
   - Database: `banking_db`

#### Step 2: 本地构建

```bash
cd /Users/chrischen/Desktop/bank_project/banking_backend
./gradlew clean bootJar
```

#### Step 3: 上传到 EC2

```bash
scp -i your-key.pem \
    build/libs/banking-0.0.1-SNAPSHOT.jar \
    ec2-user@your-ec2-ip:/opt/banking/
```

#### Step 4: 配置 EC2 环境变量

SSH 到 EC2 并创建配置文件:

```bash
sudo nano /etc/profile.d/banking.sh
```

添加内容（**替换为实际值**）:

```bash
export SPRING_PROFILES_ACTIVE=prod
export SPRING_DATASOURCE_URL=jdbc:postgresql://banking-rds.xxxxxx.region.rds.amazonaws.com:5432/banking_db
export SPRING_DATASOURCE_USERNAME=postgres
export SPRING_DATASOURCE_PASSWORD=your_secure_password
export JWT_SECRET=$(openssl rand -base64 32)
export SERVER_PORT=8080
```

加载配置:

```bash
source /etc/profile.d/banking.sh
```

#### Step 5: 启动应用

**前台运行（测试用）:**
```bash
cd /opt/banking
java -jar banking-0.0.1-SNAPSHOT.jar
```

**后台运行（推荐）:**

创建 systemd 服务:

```bash
sudo nano /etc/systemd/system/banking.service
```

添加内容:

```ini
[Unit]
Description=Banking System Service
After=network.target

[Service]
Type=simple
User=ec2-user
WorkingDirectory=/opt/banking
EnvironmentFile=/etc/profile.d/banking.sh
ExecStart=/usr/bin/java -jar /opt/banking/banking-0.0.1-SNAPSHOT.jar
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务:

```bash
sudo systemctl daemon-reload
sudo systemctl start banking
sudo systemctl enable banking
sudo systemctl status banking
```

#### Step 6: 验证部署

```bash
# 在 EC2 上测试
curl http://localhost:8080/api/accounts

# 从外部测试（需配置安全组）
curl http://your-ec2-public-ip:8080/api/accounts
```

## 🔑 关键配置说明

### 环境变量优先级

1. **最高优先级**: 系统环境变量
2. **中等优先级**: Profile 配置文件 (`application-{profile}.yml`)
3. **最低优先级**: 通用配置文件 (`application.yml`)

### Profile 选择

| Profile | 用途 | 数据库 | 日志级别 | DDL 模式 |
|---------|------|--------|----------|----------|
| `local` | 本地开发 | 本地 PostgreSQL | DEBUG | update |
| `prod` | 生产环境 | AWS RDS | INFO | validate |

### 重要配置项

1. **SPRING_PROFILES_ACTIVE**
   - 值: `local` 或 `prod`
   - 作用: 决定加载哪个配置文件

2. **SPRING_DATASOURCE_URL**
   - 本地: `jdbc:postgresql://localhost:5432/banking_db`
   - 生产: `jdbc:postgresql://your-rds-endpoint:5432/banking_db`

3. **JWT_SECRET**
   - ⚠️ 生产环境必须使用安全密钥！
   - 生成方法: `openssl rand -base64 32`

4. **HIBERNATE_DDL_AUTO**
   - 本地: `update` (自动更新表结构)
   - 生产: `validate` (仅验证，不修改)

## 🚨 重要提示

### 安全注意事项

1. **永远不要在代码中硬编码敏感信息**
   - ❌ 密码、密钥不要写在配置文件中
   - ✅ 使用环境变量

2. **生产环境必须更换默认密钥**
   - JWT Secret
   - 数据库密码

3. **限制数据库访问**
   - RDS 安全组只允许 EC2 访问
   - 不要使用 `0.0.0.0/0`

4. **使用 HTTPS**
   - 配置 Nginx 反向代理
   - 使用 SSL 证书

### 数据库操作注意事项

1. **本地开发**
   - 可以使用 `ddl-auto=update`
   - 测试数据可以随时删除重建

2. **生产环境**
   - 使用 `ddl-auto=validate`
   - 禁止自动修改表结构
   - 做好数据备份
