## 🌐 部署到 EC2

### 快速部署流程

1. **本地构建**
   ```bash
   ./gradlew clean bootJar
   ```

2. **上传到 EC2**
   ```bash
   scp -i your-key.pem \
       build/libs/banking-0.0.1-SNAPSHOT.jar \
       ec2-user@your-ec2-ip:/opt/banking/
   ```

3. **在 EC2 上配置环境变量**
   ```bash
   # SSH 到 EC2
   ssh -i your-key.pem ec2-user@your-ec2-ip
   
   # 创建配置文件
   sudo nano /etc/profile.d/banking.sh
   ```
   
   添加（替换为实际值）：
   ```bash
   export SPRING_PROFILES_ACTIVE=prod
   export SPRING_DATASOURCE_URL=jdbc:postgresql://your-rds-endpoint:5432/banking_db
   export SPRING_DATASOURCE_USERNAME=postgres
   export SPRING_DATASOURCE_PASSWORD=your_password
   export JWT_SECRET=$(openssl rand -base64 32)
   ```
   
   加载配置：
   ```bash
   source /etc/profile.d/banking.sh
   ```

4. **启动应用**
   ```bash
   cd /opt/banking
   ./start-prod.sh
   ```

## ❓ 常见问题

### 问：本地无法连接数据库？

**答：** 确保 PostgreSQL 正在运行：

```bash
# 检查状态
brew services list  # macOS
sudo systemctl status postgresql  # Linux

# 启动 PostgreSQL
brew services start postgresql  # macOS
sudo systemctl start postgresql  # Linux

# 测试连接
psql -U postgres -d banking_db
```

### 问：端口 8080 被占用？

**答：** 查找并杀死占用端口的进程：

```bash
# 查找进程
lsof -i :8080

# 杀死进程
kill -9 $(lsof -ti:8080)

# 或更换端口
export SERVER_PORT=8081
```

### 问：如何查看应用日志？

**答：**

```bash
# 本地开发（直接在终端查看）

# 生产环境（systemd）
sudo journalctl -u banking -f

# 生产环境（文件日志）
tail -f /var/log/banking/application.log
```

### 问：如何重新创建数据库？

**答：**

```bash
# 删除并重新创建（会丢失所有数据！）
psql -U postgres -c "DROP DATABASE banking_db;"
psql -U postgres -c "CREATE DATABASE banking_db;"

# 重启应用（会自动创建表）
./start-local.sh
```

## 🔒 安全提醒

### ⚠️ 生产环境必须做的事：

1. **更换 JWT 密钥**
   ```bash
   export JWT_SECRET=$(openssl rand -base64 32)
   ```

2. **使用强密码**
   - 数据库密码至少 16 字符
   - 包含大小写字母、数字、特殊字符

3. **限制数据库访问**
   - RDS 安全组只允许 EC2 访问
   - 不要使用 `0.0.0.0/0`

4. **配置 HTTPS**
   - 使用 Nginx 反向代理
   - 配置 SSL 证书
